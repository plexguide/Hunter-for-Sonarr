General Settings

The General Settings section allows you to configure basic options for your Huntarr.io instance.

System Settings

Application Name

You can customize the name of your Huntarr.io instance that appears in the title bar and navbar. This is useful if you're running multiple instances.

Default: Huntarr.io

UI Theme

Choose between different UI themes for your Huntarr.io interface:

  • Light: Default light theme
  • Dark: Dark theme for low-light environments
  • System: Follows your system's dark/light mode settings

Language

Select the language for the Huntarr.io UI. Current supported languages include:

  • English (Default)
  • Spanish
  • French
  • German
  • Italian
  • More languages are being added regularly

Updates

Update Channel

Select which update channel to use:

  • Stable: Recommended for most users. Includes thoroughly tested updates.
  • Beta: Preview newer features before they reach stable. May contain bugs.
  • Nightly: Latest development builds. Not recommended for production use.

Automatic Updates

Configure how Huntarr.io handles updates:

  • Automatic: Huntarr.io will automatically update when new releases are available.
  • Notify Only: You'll be notified about updates but they won't be installed automatically.
  • Disabled: No update checks will be performed.

Backup

Backup Location

Specify where backups should be stored. Default location is /opt/appdata/huntarr/backups, but you can change this to any accessible path.

Backup Schedule

Configure automatic backups of your Huntarr.io configuration:

  • Daily: Run backups every day (recommended)
  • Weekly: Run backups once per week
  • Monthly: Run backups once per month
  • Disabled: Don't run automatic backups

Backup Retention

Determine how many backups to keep before deleting older ones:

  • Default: 7 backups
  • Range: 1-30 backups

Logging

Log Level

Select how detailed the logs should be:

  • Error: Only log errors
  • Warning: Log errors and warnings
  • Info: Default, includes general operation information
  • Debug: Detailed logs, useful for troubleshooting but generates larger log files
  • Trace: Most detailed logging level, should only be used when debugging issues

Log Rotation

Configure when logs should be rotated:

  • Size: When log file reaches a certain size (default: 10MB)
  • Daily: Rotate logs once per day
  • Weekly: Rotate logs once per week

Log Retention

How many log files to keep before deleting older ones:

  • Default: 10 files
  • Range: 1-50 files
